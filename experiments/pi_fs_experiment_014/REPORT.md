# pi-fs experiment 014: algebraic suffix recovery from cross-base certificate

## Goal

Test the claim:

```text
If the first n decimal digits of a sector are known,
and a certificate in another base gives a sufficiently narrow sector,
then the next k decimal digits can be recovered algebraically.
```

No text search is used to select the hidden suffix. The hidden digits are derived
by proportional sector arithmetic.

For a boundary H, define:

```text
start = H - n
known prefix = pi[start : H]
hidden suffix = pi[H : H+k]
```

The phase sector begins at `start`, so the full `n+k` digit decimal cell is:

```text
A = int(pi[start : H+k])
```

A cross-base certificate in base Q at depth L is:

```text
C = floor(Q^L * {10^start pi})
```

Then translate the Q-cell to decimal depth `D = n+k`:

```text
A_min = floor(C * 10^D / Q^L)
A_max = ceil((C+1) * 10^D / Q^L) - 1
```

Intersect this range with the known prefix range:

```text
10^k P_n <= A <= 10^k(P_n+1)-1
```

If exactly one A remains, then:

```text
hidden = A - 10^k P_n
```

  the candidate suffix set collapses to exactly one value.
- This directly formalizes the idea that a certificate for a 100-digit sector is
  also a certificate for a 99-digit prefix plus an algebraically determined X.
- For `k=1`, this is exactly the "10 candidates" situation.
- For larger `k`, the candidate set has size `10^k` before certification, but
  proportional intersection with the cross-base certificate collapses it.
- Repeating `k=1` n times gives O(n) continuation rounds if an independent
  certificate is available at each step.

## Boundary

The certificate is the information source. This experiment uses already-known
future digits to simulate that independent cross-base certificate. In a real
beyond-horizon setting, the certificate must come from an interval computation,
digit oracle, or another independently verified phase source.
