# pi-fs experiment 013: boundary continuation by 10 candidates + certificate

## Goal

Test the proposed continuation contour:

```text
known_tail_100
-> unique boundary anchor
-> 10 candidate digits
-> negative-context alignment certificate
-> independent next-digit certificate
```

The experiment hides the next 100 digits after several artificial cutoffs inside
an already-computed prefix of pi. Then it reconstructs those 100 digits one by
one using the 10-candidate procedure.

## Interpretation

- The 100-digit boundary tail is unique at all tested cutoffs.
- For each future digit, exactly 10 candidates are considered.
- Negative-context alignment confirms that all candidates are attached to the
  correct boundary.
- The independent certificate accepts exactly one candidate per step.
- Repeating this procedure reconstructs the next 100 digits exactly in every
  tested cutoff.

## Important boundary

This is not a way to get new digits from nothing.

The independent certificate is the information source that selects the correct
candidate. In this backtest, that certificate is simulated by already-computed
future pi digits. In a real beyond-horizon setting, it would have to come from
an interval computation, independent digit computation, or another certified
oracle.

What the experiment confirms is the architecture:

```text
constant-size candidate set + O(tail_len) boundary alignment + independent certificate
```

For fixed 100-digit tail, extending n digits requires O(n) candidate rounds.
With tail length chosen as O(log H), the alignment part is O(n log H).
