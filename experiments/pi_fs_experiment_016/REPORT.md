# pi-fs experiment 016: external sector certificate validation

## Goal

Experimentally validate the formal lemma:

```text
known decimal prefix + independent external sector certificate
    => next digit or suffix can be extracted algebraically
```

Here "external" means external relative to the information in the known decimal
prefix. It may still be produced by ordinary mathematics: another base,
interval computation, binary/hex extraction, or any verified phase oracle.

For a boundary H:

```text
known prefix = pi[H-n : H]
hidden suffix = pi[H : H+k]
```

External certificate in base Q:

```text
C = floor(Q^L * {10^(H-n) pi})
J = [C/Q^L, (C+1)/Q^L)
```

Translate the Q-sector to decimal depth `D = n+k`:

```text
A_min = floor(C * 10^D / Q^L)
A_max = ceil((C+1) * 10^D / Q^L) - 1
```

Intersect with the known prefix range:

```text
10^k P_n <= A <= 10^k(P_n+1)-1
```

If exactly one `A` remains:

```text
hidden = A - 10^k P_n
```

## Interpretation

- Without an external certificate, one next digit has 10 candidates and a
  k-digit suffix has 10^k candidates.
- A sufficiently deep external sector certificate collapses the candidate set to
  exactly one value.
- For the one-digit case with known_n=100, base-16 certificates select the next
  digit at lower offset than base-4 or base-2 because their sectors shrink faster.
- For block recovery, the same algebra works: a sufficiently narrow external
  sector determines the whole hidden block.

## Boundary

This validates the algebraic extraction/selection mechanism. It does not create
the external certificate. In a beyond-horizon setting, the certificate must come
from an independent interval computation, digit-extraction method, or verified
phase source.
